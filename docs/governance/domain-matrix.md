# D00–D12 Domain Matrix

> 狀態：Draft
> 架構決策日：2026-07-18
> 適用範圍：臺灣地區之機電工程與起重工程 Compliance & Code of Conduct 文件系統

## 1. 決策

本專案採 D00–D12 作為長期、不可任意重編的控制領域。實體文件以 Domain 組織；Core、Role-Based、Conditional 與 Local Addendum 是適用維度，不是第一層目錄。

本文件是 Domain taxonomy、責任邊界與初始控制矩陣的唯一來源。README 只提供摘要。

## 3. 永久邊界規則

1. D00–D12 發布後不重新排序、交換或回收。
2. 顯示名稱可以因語言或組織用語微調，但 Primary Responsibility 不得偷換。
3. 新議題先作為既有 Domain 的模組；只有出現全新控制目的與可識別的獨立 Owner 時，才新增 D13 以後的 Domain。
4. 每個模組只有一個 Primary Domain。其他 Domain 以 related module、procedure 或 evidence reference 連結。
5. D00 是全員共同基準，不取代各 Domain 對所有人適用的細部控制。
6. Repository 文件治理、法規台帳與模板放在 `docs/governance/`，不算另一個 CoE Domain。
7. 工地、業主、契約及所在地規則放在 `docs/addenda/`，不建立永久 Domain。

## 4. Applicability 維度

| Type | Selector | Required Evidence |
| --- | --- | --- |
| Core | 指定人口持續適用 | 人口定義、發布／納入方式、版本與生效日 |
| Role-Based | 職務、責任、法定資格或持續權限 | 角色指派、資格、授權、交接與撤銷紀錄 |
| Conditional | 任務、資產、契約、地點、資料或臨時權限 | trigger、開始日、終止／撤銷條件及佐證 |
| Local Addendum | 工地、業主、契約或所在地 | 適用範圍、期間、核准人及衝突處理 |

一個模組可以有多筆 applicability rule。Role-Based 與 Conditional 可以同時適用：吊裝主管的持續責任回答 `who`；某次吊裝任務、設備、地點與許可則回答 `when`。每筆 rule 必須分別記錄 selector 與 evidence。

每筆 rule 使用模組內唯一的 `AR-nnn`，並以 `applies_to` 綁定 Requirement IDs。規則內 selectors 預設為 AND；不同 rules 預設以 OR 判斷模組適用；同時命中時，Requirement IDs 採聯集。人員指派與稽核紀錄必須保存 `Module ID + Rule ID`。

## 6. 關鍵跨域邊界

### 6.1 財務、商務與採購

- D03 管付款、帳務與財務職務分離。
- D04 管 sell-side 商務定價、對外報價、投標價格、客戶及市場行為。
- D08 管 buy-side 採購、供應商與承攬商遴選。
- D05 管技術數量、資源需求與工程成本估算；其輸出可供 D04 定價，但不等於對外價格承諾。

### 6.2 工程、品質、職安與設備

- D05 決定工程方法、設計與專案交付。
- D06 定義量測與校正要求，獨立驗證檢驗、測試、校正與驗收證據。
- D07 管作業危害、許可、共同作業與停止作業。
- D10 管設備、機具、量測工具、吊具與車輛的送校、標示、隔離、維護、復役及生命週期狀態。

### 6.3 承攬與第三方

- D08 管第三方遴選、盡職調查與契約條件。
- D05 管承攬工作與工程介面的交付整合。
- D07 管進場後的危害告知、共同作業與安全協調。

### 6.4 資訊與 AI

- D11 管身分、存取、資料、個資、營業秘密與基礎資安。
- D12 管 AI 准用、輸入與輸出風險、人工覆核、自動化及演算法決策。

## 7. Module ID 規則

- 格式為 `Dxx-Mnnn`，例如 `D07-M001`。
- Module ID 只表示 Primary Domain 與永久流水號。
- Core、Role-Based、Conditional、職稱、設備與地點不得寫入 Module ID。
- 流水號只在模組責任邊界獲核准後分配。
- 退役 Module ID 不回收；須保留狀態與替代文件引用。

## 8. Domain Matrix 最低欄位

未來每個已核准模組至少記錄：

| Field | Requirement |
| --- | --- |
| Module ID | `Dxx-Mnnn` |
| Primary Domain | D00–D12 中唯一一項 |
| Title | 不依賴暫時性部門名稱 |
| Responsibility Boundary | 包含內容與明確排除事項 |
| Applicability Rules | 一或多筆具 `AR-nnn`、selector、evidence 與 Requirement ID mapping 的 rule |
| Audience / Role | 適用人口、角色、資格或權限 |
| Trigger / End Evidence | 啟用、終止與撤銷佐證 |
| Owner / Approver | 內容與核准責任人 |
| Legal Evidence IDs | 法規台帳引用，不直接複製大量法條 |
| Procedures / Records | 執行方式與證據位置 |
| Status / Version | 文件狀態、版本與有效期間 |
